# 2026-06-04_1600 — Fully-offline Jina model vendoring

The JinaBERT modeling code is now vendored into the model folder so loading
never needs the network.

## What was done

- Downloaded `configuration_bert.py` (8.5 KB) and `modeling_bert.py` (96 KB)
  from `jinaai/jina-bert-v2-qk-post-norm` on HF hub into
  `backend/models/jina-embeddings-v2-base-code/`.
- Patched `config.json` `auto_map` to drop the `jinaai/jina-bert-v2-qk-post-norm--`
  repo prefix from all four entries so `transformers` resolves the classes
  locally rather than fetching from the hub.
