# 2026-06-11 12:00 — Enforce two environments + runnable migration in the AppNova agents

Goal: make EVERY uploaded app's conversion automatically produce (1) both a
dev and a prod environment and (2) a complete runnable migration (schema
applies on startup, dev path seeds) — i.e. bake this session's manual fixes
into the generator so they happen by default.

## Changes (AppNova product source)
- **`backend/agents/prompts.py`** (code-generation prompt): added two
  non-negotiable sections —
  - **TWO ENVIRONMENTS (dev + prod)** — per-stack requirements (.NET
    appsettings.{Development,Production}.json incl. `Cors:AllowedOrigins`;
    Node `.env.development`/`.env.production`; Python per-env settings).
  - **COMPLETE RUNNABLE MIGRATION** — schema must auto-apply on first run
    with zero manual steps. Includes the exact .NET provider-branch startup
    (MigrateAsync on SQL Server / EnsureCreatedAsync + DevSeeder on SQLite),
    the design-time-factory pattern for emitting a SqlServer InitialCreate
    migration, and idiomatic equivalents for Prisma/Drizzle/Django/Alembic.
- **`backend/agents/auditors/env_migration.py`** (new): deterministic,
  stack-aware (.NET/Node/Python) post-codegen auditor. Verifies both
  environments are present and the migration is wired to apply on startup
  with a dev seed; writes `docs/ENV_MIGRATION_AUDIT.md`; never raises;
  returns a manifest. Advisory (surfaces issues; supervisor decides on any
  downgrade), matching seed_completeness/audit_run_scripts.
- **`backend/main.py`**: import + call `audit_env_migration(converted_dir)`
  in the post-generation auditor chain (after run_scripts), surfacing an
  `env_migration_audit` manifest over SSE.
- **`README.md`**: added env_migration to the "After-the-agents auditors"
  table and the auditors file-tree listing.
