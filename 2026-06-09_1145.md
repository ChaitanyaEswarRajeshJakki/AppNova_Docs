# 2026-06-09 11:45 — Correct README deploy-details field count (24 → 14)

## Problem
The README described the Deployment Details panel as a "24-field form
incl. Azure AD GUIDs / Key Vault refs / SMTP / SSO". The actual UI form
(`frontend/index.html`, `frontend/app.js` `deployConfigPanel`) only exposes
14 fields and has no inputs for Azure AD / Key Vault / SMTP / SSO.

The "24" comes from the agent-facing config schema
(`backend/agents/example_deploy_config.json`), which has 24 keys. The README
conflated that JSON schema with the UI form. Pre-existing doc drift — not
introduced by any recent code change.

## Fix
`README.md` — three spots corrected to match reality:
- Feature bullet (placeholder hardening): form is 14 fields; the extra 10
  Secrets & SSO keys live only in `context/deploy_config.json`.
- API table (`/api/session/{id}/deploy-config`): form exposes 14 fields,
  schema accepts 10 more not yet in the form.
- Quick-start step 4: lists the real 14 form fields; notes Secrets & SSO
  keys are config-file-only for now.
