v2 Claude Code CLI Runner — module map

The modules below replace the central column's LangGraph + 4-provider pipeline when the user hits /api/analyze-v2 or /api/chat. Each agent runs as a claude -p subprocess under the Claude Code Max subscription — no API keys consumed on the happy path. The v1 pipeline remains active as a fallback.

🚀 Runner (subprocess)
agents/runner.py
Threaded subprocess.Popen pumps stream-json lines via loop.call_soon_threadsafe. Works on both Selector & Proactor loops. Windows .cmd shim bypassed via node.exe cli.js. 3-attempt transient retry; CREATE_NO_WINDOW.
🧭 Supervisor (DAG waves)
agents/supervisor.py
compute_waves() topological sort over AgentSpec.upstream. Wave 0 = 9 parallel agents · Wave 1 = code-generation · Wave 2 = testing + ui-ux. Per-cwd lock serialises writers.
🔍 Discovery + briefs
agents/runner.run_discovery · prompts.split_discovery_into_briefs
One cheap Haiku pass writes context/digest.md + brief_<agent>.md so each specialist skips re-exploration. ~2 min on a 300+ file monolith.
📁 Session adapter
agents/session_adapter.py
Mirrors uploads/{sid}/files/ → source/ · detects project root (package.json, pom.xml, composer.json, .csproj, go.mod, ...) · creates converted/ + context/.
💬 Chat bridge
agents/chat_bridge.py
claude -p --output-format json with Read/Glob/Grep allowed · Write/Edit/Bash denied. Never raises → /api/chat falls back silently to v1 chat_graph on error.
⚙️ Registry & prompts
agents/config.py · agents/prompts.py
12-agent AGENT_REGISTRY with tier + upstream deps; tier→model map (HEAVY_MODEL=sonnet-4-6, LIGHT_MODEL=haiku-4-5). 24h default timeouts.
💰 Cost ledger
core/cost_tracker.py + model_pricing.yaml
Per-call SQLite ledger (data/cost_tracking.db) with openpyxl 5-sheet workbook. Subscription absorbs actual spend; ledger shows API-equivalent virtual cost.
📋 Report combiner
agents/v2_to_report_data.py
Concatenates per-agent markdown results into the single-string shape tools/document_generator.generate_docx consumes. No export-pipeline changes required.
🔌 MCP trace server
mcp_server/trace_server.py
Local stdio MCP server exposing trace SQLite read-only. Register via claude mcp add appnova-traces so Claude Code CLI sessions query backend telemetry directly.
Event envelope — v2 SSE emits rich agent_start / assistant (with tool_calls) / tool_result / agent_complete events alongside the legacy {type: start|complete|error, agent_id} shape, so the existing api.ts frontend works unchanged (single URL flip from /api/analyze/stream → /api/analyze-v2). · Full port log + 9 hotfixes: changes.md (2026-04-21).
