### Point 11 ‚Äî Session source auto-cleanup (was: manual endpoint only)

**`backend/main.py`** ‚Äî `run_in_background()` `finally` block (~line 4067)

Added automatic `clear_session_source` call for every registered playbook immediately
after `AuditLog.session_end()`. Previously the `__source__{sid}` Chroma collection was
only cleaned up if the frontend explicitly POST-ed `/api/rag/clear-session/{sid}`.
Now it fires unconditionally on session teardown (stop, error, completion).

```python
try:
    from backend.playbooks.rag import clear_session_source
    from backend.playbooks.registry import PLAYBOOK_REGISTRY
    for _pid in PLAYBOOK_REGISTRY:
        clear_session_source(playbook_id=_pid, session_id=session_id)
except Exception:
    pass
```

### Point 13 ‚Äî Admin CLI: purge by agent_version (NEW)

**`backend/playbooks/rag/admin.py`** ‚Äî new module, three sub-commands:

| Command | What it does |
|---|---|
| `stats` | Prints `rag_status()` ‚Äî collection names + counts |
| `list-versions [--playbook P]` | Lists unique `agent_version` tags in `__learned` |
| `purge --version V [--playbook P]` | Deletes `__learned` entries where `agent_version == V` |

Run as: `python -m backend.playbooks.rag.admin <subcommand> [flags]`

Example:
```
python -m backend.playbooks.rag.admin purge --version v2026.05.13
python -m backend.playbooks.rag.admin purge --version v2026.05.13 --playbook laravel-to-dotnet
python -m backend.playbooks.rag.admin list-versions
```
