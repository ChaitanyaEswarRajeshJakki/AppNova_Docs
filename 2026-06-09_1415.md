# 2026-06-09 14:15 — Convert HTTP middleware to pure ASGI (kill the SSE disconnect noise)

## Problem
backend.log repeatedly showed:

```
RuntimeError: Unexpected message received: http.request
  ... starlette/middleware/base.py ... body_stream / receive_or_disconnect ...
```

Non-fatal (server kept serving), but noisy on every browser navigate/refresh/close while a
run-status SSE stream was open.

## Root cause
Both HTTP middlewares used `@app.middleware("http")`, i.e. Starlette's `BaseHTTPMiddleware`, which
wraps the connection's receive channel in an anyio task group. On a long-lived **streaming**
response (SSE), a client disconnect makes its disconnect poller receive an unexpected
`http.request` and raise. The capture middleware's own comment already named this exact symptom.
Converting only one wasn't enough — the **auth gate is the outermost** middleware, so every SSE
stream (`/api/resume`, `/api/run-selected`, `/api/session/{id}/events`, `/api/run/stream/...`)
flowed back out through it.

## Changes — `backend/main.py`
Rewrote BOTH middlewares as pure-ASGI classes (`async def __call__(self, scope, receive, send)`),
registered via `app.add_middleware(...)` at the same points so the stack order is unchanged
(outermost→innermost: **auth → capture → CORS → routes**). Pure ASGI only *observes* receive/send
and passes streaming responses straight through — no receive-channel wrapping, no task-group
disconnect race.

- `_HTTPCaptureMiddleware` (was `_http_capture_middleware`): behaviour preserved 1:1 — one INFO
  line per request; `/health` silent; on status ≥ 400 (non-SSE) a WARNING with request + response
  body previews capped at `_BODY_PREVIEW_LIMIT`. Request body is now *observed* as the app reads it
  (via a `receive` wrapper) instead of eagerly drained + replayed; response body is buffered only
  for non-SSE errors while every message passes through unchanged (no response rebuild).
- `_AuthGateMiddleware` (was `_auth_gate`): faithful port — auth-disabled / OPTIONS / non-`/api/` /
  public paths pass through; Bearer token required otherwise (header or `?token=` EventSource
  fallback); same 401 JSON with manual CORS headers.

No `BaseHTTPMiddleware` remains in the stack.

## Verification
- `main.py` parses + imports clean.
- Stack now: `['_AuthGateMiddleware', '_HTTPCaptureMiddleware', 'CORSMiddleware']` — no
  `BaseHTTPMiddleware`.
- Functional ASGI smoke: `GET /ok → 200` (one-liner logged); `/health` silent; `GET /boom → 400`
  with `response_body: {"detail":"bad input"}` captured; **streaming response + mid-stream
  `http.disconnect` → status 200, no crash** (previously the `Unexpected message received`
  RuntimeError).
