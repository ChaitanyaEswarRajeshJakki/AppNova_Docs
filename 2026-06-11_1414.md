# 2026-06-11 14:14 — Fix playbook mis-classification + RAG self-poisoning (hybrid Laravel+AngularJS)

## Root cause (why the latest conversion was minimal + April was never "captured")
`TotalBookingAI` is a **hybrid** app: a Laravel PHP/Eloquent backend AND an
AngularJS 1.x frontend. `resolve_playbook` mis-classified it as the
**frontend-only `angularjs-to-react`** playbook for two compounding reasons:

1. **Wrapper dir.** The project is nested under `TotalBookingAI-Input/`, so
   Laravel's path-anchored signals (`app/Models/**`, `app/Http/Controllers/**`,
   `routes/api.php`) never matched. AngularJS signals are filename globs
   (`*.controller.js`, `*.service.js`…) that survive nesting → AngularJS won
   6–3 (and still 6–5 even when descended — it's a genuine hybrid).
2. **No hybrid playbook existed.** `angularjs-to-react` is frontend-only
   (`csharp_namespace=""`, skips `data-migration`, zero backend hints), so the
   .NET backend generated with no playbook guidance → thin/minimal output.

Consequences, confirmed against the live Chroma store (292 MB):
- The only learned collection was `angularjs-to-react__learned` (177 rows) —
  **176 from the June minimal run** (`58dd9749df6d`, ~89% parity) + 1 stray;
  **zero from the good April run** (`c9afaa02f85e`). No `laravel-to-dotnet__*`
  collection ever existed.
- The automatic `_step_rag_learn` had ingested June's own minimal output as a
  "good" example (parity ≥ floor + supervisor_ok, `human_verified=False`), which
  future runs then retrieved and imitated — a self-poisoning loop. April's good
  conversion was never available to retrieve.
- Separately surfaced (not code-fixed here): `sentence-transformers` is **not
  installed**, so all embeddings/retrieval silently run on the 256-dim **hash
  fallback** ("useless for semantic search"); the on-disk jina model is unused.

## Changes
### `backend/playbooks/schema.py`
- New `PlaybookRAG.learned_require_human_verified: bool = False`.

### `backend/playbooks/registry.py`
- **Fix #1 — wrapper descent.** New `_descend_to_project_root(upload_dir)`
  (local mirror of main.py's `_detect_project_root`, skips `context/chat/…`)
  called at the top of `resolve_playbook` so path-anchored signals match.
- **Fix #2 — hybrid playbook** `laravel-angularjs-to-dotnet-react`
  ("Laravel PHP + AngularJS → .NET 8 + React 18"): full-stack, `csharp_namespace`
  set, does NOT skip `data-migration`, carries BOTH the Laravel→.NET backend
  hints AND the AngularJS→React frontend hints. Its `source_signals` are
  **exactly** the union of the two single-stack sets (verified
  programmatically), and it is registered **last** so ties break toward the
  single-stack playbooks — a pure-FE or pure-BE source keeps its single-stack
  playbook; only a genuinely full-stack source out-scores both halves.
- **Fix #4** — all four playbooks now set `rag.learned_require_human_verified=True`.

### `backend/playbooks/rag/ingestion/ingest_result.py`
- **Fix #4 — enforce the human-verified gate.** After the parity/round-trip
  gates, reject writes when `rag.learned_require_human_verified` (or env
  `APPNOVA_RAG_LEARN_REQUIRE_HUMAN=1`) is set and `human_verified` is False.
  Reason string: `"human verification required for __learned"`.

### `scripts/rag_reseed.py` (new)
- **Fix #3** — one-shot repair, dry-run by default (`--apply` to commit):
  1. **Purge** the misclassified rows from `angularjs-to-react__learned`
     (sessions `58dd9749df6d`, `3dfca6fbea95`).
  2. **Re-ingest** the verified-good April conversion (`c9afaa02f85e`,
     file_map at `context/file_map.json`, flat-layout converted root) into
     `laravel-angularjs-to-dotnet-react__learned` with `human_verified=True`.
- Uses the runtime `embed_texts`, so the re-seed stays dimension-consistent
  with retrieval (currently hash/256-dim).

### Docs
- `README.md` — registry table now lists 4 playbooks incl. the hybrid; documented
  wrapper-descent + union/tie-break ordering; updated the `__learned` invariant
  to cover the human-verified gate.
- `.env.example` — documented `APPNOVA_RAG_LEARN_REQUIRE_HUMAN`.
