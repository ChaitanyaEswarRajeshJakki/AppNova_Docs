# 2026-06-12 12:19 — Enforce frontend style fidelity: new `style_fidelity` auditor

## Goal (user)
"Every conversion should be clean and rich, with a clean frontend that mirrors
the source HTML/CSS in the target stack without missing anything."

## Diagnosis
The code-generation prompt already MANDATES frontend/CSS fidelity (Section
"Visual + layout parity" + Section A "Stylesheet pipeline must actually
compile"), but nothing ENFORCED it deterministically. `line_count_fidelity`
only reds a stylesheet that has a 1-to-1 `file_map.json` row; it never looks at
stylesheets the planner skipped/merged, and it can't see the "page renders
unstyled" wiring bugs. That's exactly how session `58dd9749df6d` (June) shipped
a frontend that ported the SCSS but still rendered unstyled/re-skinned vs the
source — measured by the new auditor at **17%** of the source's CSS
declarations with **3 global stylesheets that nothing imports**.

## Change — new deterministic auditor `backend/agents/auditors/style_fidelity.py`
`audit_style_fidelity(converted_root, source_root, context_dir)` →
`docs/STYLE_FIDELITY.md` + manifest. Checks:
- **Dropped source stylesheets** — app-owned source stylesheets (vendor/minified
  excluded) with no target counterpart (no file_map row, no stem match).
- **Zero-CSS wiring bugs**: (a) `.scss` ships but no `sass`/`sass-embedded` in
  package.json; (b) a non-partial, non-module stylesheet that no script imports
  and no stylesheet `@import`s (dead → Vite never compiles it); (c) a
  bare-package `@import`/`@use` whose npm package isn't in dependencies.
- **Bulk shrinkage** — target ships < 40% of the source's CSS declarations.

**Calibration (critical to avoid false positives on good conversions):**
- **Styling-strategy aware.** Detects Tailwind/UnoCSS/Windi targets (config
  file, dep, or `@tailwind`/`@apply`). For a `utility`-CSS target the styling
  lives in `className` utilities, so the coverage + shrinkage rows are
  downgraded to informational (the wiring checks still run). Without this the
  April Tailwind conversion — which renders correctly — false-flagged at "1%
  CSS ported".
- **Vendor/minified excluded** from the to-port source set (Kendo, Bootstrap,
  `*.min.css`, `vendor.scss`, paths under `lib/`/`vendor/`/`third-party/`) —
  they become an npm dependency, not hand-ported rules.
- **SCSS partials vs npm packages** — `@import 'variables'` resolves to a local
  `_variables.scss` (not flagged); only `@import 'bootstrap/...'` / scoped / `~`
  specifiers count as packages. Killed the false "unresolved package" noise.

Validated against the live conversions:
- **April `c9afaa02f85e`** (Tailwind) → `strategy=utility`, **clean, no warning**.
- **June `58dd9749df6d`** (SCSS) → flags 3 dead global stylesheets
  (`styles/global/generic.scss`, `styles/global/helpers.scss`,
  `pages/PCDEC/pcdecVariables.scss`), 27 dropped source stylesheets, 17% decl
  ratio; **0** false-positive unresolved packages.
