Diffs every permission string the frontend gates on against the backend source.
Splits **route-guard** permissions (`<ProtectedRoute permission=>` /
`requiredPermission` / `[Authorize(Policy=)]` — missing = unreachable PAGE) from
**UI-visibility** permissions (`allows()`/`can()`/`hasPermission()` — missing =
hidden control). Stack-agnostic: a frontend permission is "known" if its exact
string appears anywhere in the backend tree. Writes `docs/PERMISSION_PARITY.md`.
- Registered in `backend/agents/__init__.py`; dispatched in `backend/main.py`
  after `audit_ui_binding` (emits `permission_parity_audit`).
- Validated: JUNE (post-fix) 0 missing route-guards / 0 ui; APRIL 0 route-guards
  / 1 ui (`create_total_booking_juvenile` — a real hidden sidebar button, NOT a

## Part C — Booking create persistence: BUG FOUND (fix NOT yet applied)
Verified via a live backend (isolated port/DB) + direct SQLite query:
- `POST /api/total-bookings` creates a row and returns "Total Booking Saved
  Successfully" — the row DOES persist to the dev `total_bookings` table.
- **BUT ~115 of ~130 columns are dropped.** Created rows have
  `first_name/last_name/dob/pc_narrative = NULL`. Root cause is self-documented
  in the converted code, `Repositories/TotalBooking/TotalBookingRepository.cs`
  `MapFieldsToEntity` (line ~566):
  `// BEHAVIOUR-DRIFT: PHP Eloquent::create($request->all()) fills all
  mass-assignable fields. Here we populate known scalar fields explicitly…`
  `ApplyFields` maps only ~16 columns. Compounding: the service
  `TotalBookingService.CreateAsync` never copies the typed DTO scalars
  (FirstName/LastName/…) into the `fields` dict, and the DTO has no
  `[JsonExtensionData]` so the ~115 un-typed posted fields are dropped at
  deserialization.

### Proposed fix (3 parts, to implement next)
1. DTO `TotalBookingStoreRequest`: add a `[JsonExtensionData]` property to
   capture every posted field the typed params don't name.
2. Service `CreateAsync`: merge typed scalars + JsonExtensionData overflow +
   `extra_fields` into the `fields` dict before building the payload.
3. Repo: replace the ~16-field `ApplyFields` stub on the create path with a
   reflection mass-assignment over `_db.Model.FindEntityType(...).GetProperties()`
   (column-name → CLR property), JsonElement-aware value coercion, scalar EF
   properties only. This restores Laravel `$request->all()` parity.
This is a focused multi-file fix + rebuild/validate cycle — deferred so it's done
right rather than rushed.
