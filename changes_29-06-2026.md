## Order 7 ‚Äî React 18 frontend bootstrap (app.js + app.conf.js + routes.js + dashboard.js)

### New files

- `frontend/src/context/AuthContext.tsx`
  - Derived helper (order 7) providing `AuthContext` and `AuthProvider`
  - Stores JWT token in `sessionStorage`; exposes `setToken` / `setPermissions`
  - Consumed by `App.tsx` (provider) and `router/index.tsx` (consumer)

- `frontend/src/config/apiConfig.ts`
  - Port of `app.conf.js` OAuthProvider.configure() block
  - All credentials read from `VITE_*` env vars (never hard-coded)
  - Exports: `API_BASE_URL`, `CLIENT_ID`, `CLIENT_SECRET`, `GRANT_PATH`, `REVOKE_PATH`, `SCOPE`, `LOGIN_ROUTE`, `TOKEN_REFRESH_INTERVAL_MS`

- `frontend/src/main.tsx`
  - React 18 entry point; replaces AngularJS `ng-app` bootstrap
  - Wraps `<App>` in `<React.StrictMode>` + `<BrowserRouter>`

- `frontend/src/App.tsx`
  - Port of `app.js` `angular.module` + `app.run()` block
  - Provides `AuthProvider`, `SearchParamsContext` (type_params), `AppStateContext` (outageShown, beagleMsgShown)
  - Schedules JWT token refresh via `setInterval` at `TOKEN_REFRESH_INTERVAL_MS` (5720000ms) matching `fence.service.js`

- `frontend/src/router/routes.ts`
  - Full port of `routes.js` $stateProvider states ‚Üí `AppRoute[]`
  - `PUBLIC_ROUTES`: login, request_reset, password_reset, register_guest, first-login
  - `PROTECTED_ROUTES`: dashboard, users, roles, admin, total-booking, pcdec, search, alerts ‚Äî with `permission` and `deniedPermissions` fields mirroring Fence guards
  - Compound guards (e.g. `edit_total_booking AND NOT only_view_all_pcdec_and_booking`) encoded as `permission` + `deniedPermissions`

- `frontend/src/router/index.tsx`
  - React Router v6 `<Routes>` + `<ProtectedRoute>` wrapper
  - Checks `AuthContext.token` ‚Üí redirects to `/login` if absent
  - Checks `allows(permissions, permission)` + `denies(permissions, deniedPermissions[i])` ‚Üí redirects to `/dashboard` if unauthorized
  - Suspense fallback for all lazy-loaded page components

- `frontend/src/pages/Dashboard/Dashboard.tsx`
  - Port of `dashboard.js` `changeWeather()` + routes.js `alertscnt` resolve
  - Day/night theme toggle using React state (replaces direct DOM manipulation)
  - Fetches `GET /api/alerts/dashboard` on mount
