# 2026-06-05 15:30 — Run manager: suppress blank launcher window

## Problem
Clicking **Run Converted** popped up a blank, empty command-prompt window on
top of the backend + frontend windows. The launcher (`cmd /c run.bat`) was
spawned with `CREATE_NEW_CONSOLE`, which forces a new *visible* console. Because
stdout/stderr are piped to the hub, the window contained nothing — a confusing
artifact.

## Fix
Both Windows spawn sites in `backend/agents/tools/run_manager.py` now use
`CREATE_NO_WINDOW` instead of `CREATE_NEW_CONSOLE`:

| Function | Line (approx) | Old flag | New flag |
|---|---|---|---|
| `_run_one_command` | ~916 | `CREATE_NEW_CONSOLE` | `CREATE_NO_WINDOW` |
| `_launch_and_await_ready` | ~1185 | `CREATE_NEW_CONSOLE` | `CREATE_NO_WINDOW` |

`CREATE_NO_WINDOW` gives the launcher a real (windowless) console. The `start
"Title" cmd /k ...` calls inside `run.bat` each request their own
`CREATE_NEW_CONSOLE`, which is not suppressed by the parent's flag — so the
visible backend/frontend windows still open. The launcher's own blank window no
longer appears.

## Verification
`python -m py_compile backend/agents/tools/run_manager.py` passes.
