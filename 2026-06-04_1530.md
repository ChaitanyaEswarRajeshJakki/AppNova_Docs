# 2026-06-04_1530 — RAG retrieval path fallback + Jina v2 local model precedence

Two changes that were documented in the change log but never applied to the actual source files.

## Fix: retrieval path fallback for curated/learned hits

**File:** `backend/playbooks/rag/retrieval/retriever.py`

Both the dedup key and `Hit.path` now fall back through `source_path → target_path` when `path` is absent:

```python
meta.get("path") or meta.get("source_path") or meta.get("target_path") or ""
```

Previously both spots used only `meta.get("path", "")`, so curated and learned entries (which carry `source_path`/`target_path`, not `path`) always returned empty paths. This caused:
- All curated hits of the same kind to dedup down to a single entry
- `must_match_path` checks in the eval harness to always fail → 0% recall on RAG-on

## Fix: Jina v2 local model takes precedence over OpenAI

**File:** `backend/playbooks/rag/embeddings/embedder.py`

- `_resolved_st_model()` — new helper that resolves `APPNOVA_RAG_ST_MODEL` relative to the repo root; defaults to `backend/models/jina-embeddings-v2-base-code`
- `_provider()` — local ST model now wins over OpenAI (was OpenAI-first)
- `_embed_st()` — loads with `trust_remote_code=True` (required by Jina v2 JinaBERT custom code); logs the loaded model path
- `provider_info()` — now reports `st_model` (resolved path) and `st_model_is_local` (bool)
