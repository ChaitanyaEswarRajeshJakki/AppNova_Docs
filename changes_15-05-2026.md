## Decisions made

### 1. Vector store: ChromaDB (not FAISS)

Conversion *results* will carry rich metadata (playbook id, source path, agent that produced it, parity score, human-verified flag, agent version). Chroma's metadata filtering (`where={...}`) and persistent collection model beat FAISS for this use case. Local persistent client now; can move to client/server mode later without code changes.

### 2. Three collections per playbook (not one)

| Collection | Source | Lifetime | Trust |
|---|---|---|---|
| `{playbook}__source` | Chunks of the user's uploaded code | Per-session, deleted on session close | Read-only context |
| `{playbook}__curated` | Hand-authored source→target pairs (20–50 per playbook) | Permanent, version-controlled | High — retrieved first |
| `{playbook}__learned` | Auto-stored verified conversions | Permanent, gated writes | Medium — fallback after curated |

Retrieval order is **curated → learned → source-siblings**. Without this split, noisy similarity hits from `learned` would shadow the gold examples.

### 3. Where RAG plugs in — *not* the parity layer

Retrieval feeds into the existing `prompt_context_for(agent_id)` hook at [backend/playbooks/schema.py:210](backend/playbooks/schema.py#L210). The deterministic mapping/parity rules in [backend/playbooks/registry.py](backend/playbooks/registry.py) (type tables, synonym pairs, ignored-field regexes) remain authoritative. RAG is a **prompt-context source**, not a replacement for `field_extractor` / `file_coverage` / `parity_checker`.

### 4. What we deliberately do NOT do

- **Not** RAG over framework docs (Laravel/.NET/React) — Claude already knows them; negative ROI.
- **Not** use vector similarity for parity gates — parity must stay deterministic and audit-friendly.
- **Not** write failed/discarded conversions into `__learned` — corpus poisoning risk.

### 5. Write-gating (non-negotiable)

A conversion is written to `{playbook}__learned` **only if** all hold:
- `parity_pct >= pb.validation.parity_green_floor_pct`
- `round_trip_passed` is true OR `pb.validation.require_round_trip` is false
- supervisor logged no fatal errors

Every row carries `{playbook_id, source_hash, target_hash, parity_pct, agent_id, agent_version, round_trip_passed, human_verified, created_at}` so a regressing agent's batch can be purged with `delete where agent_version = "vYYYY.MM.DD"`.

### 2026-05-15 — Precursor fixes P-0 / P-1 / P-2 (LANDED)

- `backend/agents/supervisor.py` — imports `resolve_playbook`; calls it once at the top of `run_supervised` against `project_dir`; threads the resolved playbook into every `build_agent_prompt` call. Best-effort: detection failure logs and falls back to `playbook=None`.
- `backend/playbooks/schema.py` — `PlaybookTransformation` docstring updated; no longer says "build_agent_prompt does not yet consume these fields directly".

**Verified end-to-end:**

**Why this matters:** every existing playbook hint (Eloquent→EF, Blade→TSX, `$scope`→`useState`, lifecycle→hooks) is now live. No new prompts authored — the dead hints simply turn on.

**Problem:** [prompts.py:359-379](backend/agents/prompts.py#L359-L379) said *"You may NOT produce un-mapped target files."* The codegen agent was contractually capped at `mappings[].targets[]` — the 28 missing files in session `c9afaa02f85e` (helpers, derived entities, `Migrations/`, `Options/`, `e2e-tests/`) were forbidden by design.

**Why this matters:** closes the 28-file gap. Derived helpers, repository utilities, EF migration files, options classes, custom hooks, contexts, module-scoped stylesheets now emit as a normal part of conversion rather than triggering "this file isn't in the map, skip it" reasoning.

#### P-2a — `synthesize_and_persist` augments instead of refuses

**Problem:** [synthesize_file_map.py:411-413](backend/agents/synthesize_file_map.py#L411-L413) refused to overwrite an existing `file_map.json`. When the planner emitted a short map (a documented failure mode — single LLM turn with no deterministic cross-check), every legitimate source file the planner missed was permanently invisible to downstream agents.

**Changed file:**

- `backend/agents/synthesize_file_map.py` — `synthesize_and_persist` rewritten: when an existing file is present, parse it, walk the disk-derived mappings, and add any disk row whose `source` path is not already in `existing_mappings[].source`. Planner-authored rows are preserved untouched; only gaps get filled. Each augmented row is tagged `_disk_inferred: True` so reviewers can distinguish auto-added rows from authoritative ones. The merged file gets `_augmented_at` + `_augmented_added` metadata.

**Why this matters:** the planner's row count stops being a hard ceiling. Files the planner missed but exist on disk get a `_disk_inferred=True` row that codegen and the parity checker can act on.

### 2026-05-15 — Observability + chunk-size revisit (LANDED)

After landing M1-M5, the user pushed back on two things: (1) was `APPNOVA_CODEGEN_CHUNK_SIZE=30` too conservative? (2) logs, payload responses, and errors should be visible without SSH'ing to the host.

#### Chunk size: 30 → 50

**Reasoning:** 30 was the audit's conservative number ("~30"). The real concern was the 80-row chunk hitting the per-turn output ceiling (≈ 6k tokens/row × 80 = 480k tokens, past every model's single-turn budget). 50 × 6k = 300k tokens — comfortably inside the ceiling **and** saves ~4 min of pure cooldowns on a 240-file migration vs 30. Cross-chunk dependencies don't break at any size in `[30, 60]` because the chunker sorts by `order` ascending and each chunk's files land on disk before the next chunk runs (the next chunk's agent uses Read).

**Changed files:**

- `backend/agents/supervisor.py` — `_CODEGEN_CHUNK_SIZE` env-var default `"30"` → `"50"`. Comment block records the trade-off (30 added cooldowns without measurable quality win).
- `backend/agents/codegen_multipass.py` — `_DEFAULT_CHUNK_SIZE = 30` → `50`. Module docstring updated.
- `.env` — `APPNOVA_CODEGEN_CHUNK_SIZE=30` → `50` (operator override now matches the new default).

#### HTTP request/response capture middleware

**Problem:** the user couldn't see what payload caused a 4xx/5xx. Backend logs only had the request URL; the body was lost once the request returned.

**Changed file:**

- `backend/main.py:88` (immediately after CORS) — added a `@app.middleware("http")` function `_http_capture_middleware`. Logs **every** request as a one-line INFO `[http] METHOD path -> status (Nms)`. On `status >= 400`, additionally logs request body + response body (each truncated to `_BODY_PREVIEW_LIMIT = 2048` bytes) as a WARNING line. Skips body capture for `multipart/form-data` (file uploads — would blow up the log) and `text/event-stream` (SSE — huge by design). Replays the request body for downstream handlers via the `request._receive` shim so the route handler still sees the original payload.

**Verified end-to-end** via `fastapi.testclient`:

```text
GET /api/logs/backend -> 200 (40 ms)            (one-line INFO)
GET /api/logs/agents  -> 200 (12 ms)            (one-line INFO)
[http-error] GET /api/this-does-not-exist -> 404 (1 ms)
  request_body:  <empty>
  response_body: {"detail":"Not Found"}         (multi-line WARNING with payloads)
```

## Per-agent audit — 2026-05-15

Pre-implementation audit by general-purpose research agent. Findings drove the P-0/P-1/P-2 fixes above.

### Finding 1 — Codegen produces too few files

**Root cause:** [prompts.py:359-379](backend/agents/prompts.py#L359-L379) hard-bound codegen to `file_map.json`; the contract forbade un-mapped target files. The planner's row count was a ceiling.
**Status:** **FIXED by P-1** (DERIVED_TARGETS register now expected, not exceptional).

### Finding 2 — Migration-planner file_map JSON is incomplete

**Root cause:** Single LLM turn, no deterministic ground-truth cross-check. [synthesize_file_map.py:411-413](backend/agents/synthesize_file_map.py#L411-L413) had a disk-walking synthesizer that refused to overwrite, so short maps never got augmented.
**Status:** **FIXED by P-2a** (synthesize now augments instead of refusing).
**Residual:** the planner itself is still a single-turn unbounded emit. A deterministic union (planner rows ∪ field_extractor ∪ source_routes ∪ stylesheet walker) at the supervisor level remains future work.

### Finding 3 — SCSS / frontend modules dropped

**Root cause:** [registry.py:32-42](backend/playbooks/registry.py#L32-L42) — Laravel playbook had no styling globs; planner had no row template for stylesheets.
**Status:** **FIXED by P-2b** (globs added across all 3 playbooks + planner hint mandates one row per stylesheet).

### Finding 4 — Output reads like source (too-literal translation)

**Root cause (compound):**

1. [prompts.py:397-399](backend/agents/prompts.py#L397-L399) explicitly mandates verbatim preservation for `1-to-1 port` rows.
2. The playbook's modernization hints (Eloquent→EF, Blade→TSX, `$scope`→`useState`) were **never read** — `build_agent_prompt` took no `playbook` argument; `supervisor.py:755` didn't pass one.

**Status:** **PARTIALLY FIXED by P-0** (hints now reach the agent; PLAYBOOK GUIDANCE block declares the playbook rule wins for framework idioms, preservation wins for domain logic).
**Residual:** still no few-shot translation pairs in prompts (Eloquent class → EF entity examples, Blade → JSX examples). That's the highest-leverage *content* improvement and is what the RAG tickets (APN-3 curated examples, APN-7 retrieval) were designed to deliver. With P-0 wired, RAG injection now has a place to land.

## RAG implementation — APN-1 through APN-15 (delivered 2026-05-15, evening round)

Implements the v1 scope described in `PLAYBOOK_RAG_DESIGN.md`: ChromaDB single store, three collections per playbook (`__source`, `__curated`, `__learned`), source-and-result ingestion, retrieval wired into code-generation + migration-planner prompts, write-gating after parity/round-trip, per-playbook feature flag.

### APN-1 — Schema: `PlaybookRAG` dataclass + `PlaybookDefinition.rag` field

- New `PlaybookRAG` (Layer 6) at [backend/playbooks/schema.py](backend/playbooks/schema.py) with fields: `enabled`, `enabled_agent_ids`, `top_k`, `retrieval_order`, `max_context_tokens`, `learned_parity_floor_pct`, `learned_require_round_trip`.
- `PlaybookDefinition` now carries `rag: PlaybookRAG = PlaybookRAG()` — default off so unchanged playbooks behave exactly as before.
- All three shipped playbooks opt-in at [backend/playbooks/registry.py](backend/playbooks/registry.py) with `enabled=True`, `enabled_agent_ids=("code-generation", "migration-planner")`, `top_k=3`. Per-playbook `learned_parity_floor_pct` mirrors each playbook's own parity floor (80 / 75 / 85).

### APN-2 — Chroma persistent-client wrapper

- [backend/playbooks/rag/client.py](backend/playbooks/rag/client.py) — process-wide `PersistentClient` singleton at `<repo>/chroma/` (override via `APPNOVA_RAG_DIR`).
- `is_available()` reports raw capability (chromadb importable + `APPNOVA_RAG_ENABLED != 0`). Per-playbook `rag.enabled` is checked separately by callers.
- Collection naming: `{playbook}__curated`, `{playbook}__learned`, `{playbook}__source__{session_id}` — per-session isolation prevents cross-tenant code leakage in `__source`.
- Telemetry disabled (`anonymized_telemetry=False`) so Chroma's `posthog` calls don't spam stdout.
- `rag_status()` returns capability + collection inventory for the `/api/rag/status` endpoint.

### APN-3 — Embeddings wrapper (3-tier fallback)

- [backend/playbooks/rag/embeddings.py](backend/playbooks/rag/embeddings.py).
- Tier 1: OpenAI `text-embedding-3-small` when `openai` is importable AND `OPENAI_API_KEY` is set.
- Tier 2: `sentence-transformers` (`BAAI/bge-small-en-v1.5`, override via `APPNOVA_RAG_ST_MODEL`).
- Tier 3: hash-based deterministic fallback. Gives Chroma a stable vector space so retrieval still works (poorly, but deterministically) for smoke tests and offline development.

### APN-4 — Per-symbol code chunker

- [backend/playbooks/rag/chunker.py](backend/playbooks/rag/chunker.py).
- Language-aware header regexes for Python / PHP / JS / TS / TSX / JSX / C# / interfaces / records / arrow-fn consts.
- Brace-walker that respects string literals (single/double/backtick + escape); indent-walker for Python.
- Files with no symbol matches (stylesheets, blade templates, configs, HTML) fall back to **whole-file chunks** so they still land in the corpus — the v1 audit found stylesheets vanishing was the worst regression, this guarantees they're preserved.
- Very large files (>24 k chars) get window-chunked. Per-chunk `metadata` carries `path`, `language`, `symbol_kind`, `symbol_name`, `file_hash`, `anchor_line`, `size`.
- `walk_and_chunk(root, max_files=5000)` skips `node_modules`, `vendor`, `.git`, `__pycache__`, `_appnova_legacy_runtime`.

### APN-5 — Per-session source ingestion

- [backend/playbooks/rag/ingest_source.py](backend/playbooks/rag/ingest_source.py).
- Walks the upload dir, batches 64 chunks at a time, calls `embed_texts` + `coll.upsert`.
- Returns `{"ingested": N, "skipped": bool, "reason": str|None}` — never raises. Failures degrade to "no RAG enrichment for this session" but never crash the pipeline.
- `clear_session_source()` companion: best-effort cleanup hook.

### APN-6 — Curated seed JSONL + `seed.py`

- [backend/playbooks/examples/laravel-to-dotnet.jsonl](backend/playbooks/examples/laravel-to-dotnet.jsonl) — 8 hand-authored source→target pairs (Model, Controller, migration, Blade view, SCSS, route table, FormRequest validator, config).
- [backend/playbooks/examples/angularjs-to-react.jsonl](backend/playbooks/examples/angularjs-to-react.jsonl) — 7 pairs (controller, service, directive, ng-template, ui-router, SCSS, auth interceptor).
- [backend/playbooks/examples/react-upgrade.jsonl](backend/playbooks/examples/react-upgrade.jsonl) — 5 pairs (class-to-hooks, PropTypes-to-TS, HOC-to-hook, Redux-to-Zustand, CSS-to-SCSS).
- [backend/playbooks/rag/seed.py](backend/playbooks/rag/seed.py) — idempotent rebuild (`seed_curated(playbook_id, replace=True)` deletes + repopulates). `seed_all()` walks every playbook in the registry.

### APN-7 — Retrieval API

- [backend/playbooks/rag/retrieve.py](backend/playbooks/rag/retrieve.py).
- `retrieve(query, *, playbook_id, session_id, top_k, kinds, where)` queries each kind in order, dedupes hits by `(kind, path, symbol_name)`, sorts `curated > learned > source` then by score.
- `format_hits_for_prompt(hits, max_chars)` renders the markdown block with language-tagged fences (tsx, ts, csharp, php, scss).
- `context_for_agent(agent_id, *, playbook, session_id, query)` is the high-level entry the prompt builder uses — honours `rag.enabled`, `enabled_agent_ids`, `top_k`, `retrieval_order`, `max_context_tokens`. Returns `""` when disabled / no hits.

### APN-8 — Gated `__learned` ingestion

- [backend/playbooks/rag/ingest_result.py](backend/playbooks/rag/ingest_result.py).
- Hard gates before writing: `parity_pct ≥ learned_parity_floor_pct`, `round_trip_passed` (if `learned_require_round_trip`), `supervisor_ok`.
- Tags every entry with `playbook_id`, `source_hash`, `target_hash`, `parity_pct`, `agent_id`, `agent_version` (`APPNOVA_AGENT_VERSION`, defaults to `v2026.05.15`), `round_trip_passed`, `human_verified`, `created_at`, `session_id`. Lets a bad batch be purged via `coll.delete(where={"agent_version": "v2026.05.13"})`.
- Returns `{"stored": bool, "reason": str|None, "id": str|None}` — never raises (corpus quality > callsite signal).

### APN-9 + APN-10 — Wire retrieval into agent prompts

- [backend/agents/prompts.py](backend/agents/prompts.py): `build_agent_prompt` gained `session_id: str | None = None` and `rag_query: str | None = None` parameters. After the `PLAYBOOK GUIDANCE` block and before `YOUR TASK`, it calls `context_for_agent` and (when non-empty) appends a `RETRIEVED EXAMPLES (curated + verified prior conversions + source siblings)` section.
- Default query is composed by `_build_default_rag_query(agent_id, playbook, target_stack, briefs_hint)` — playbook label + source/target tech + agent_id + truncated target_stack + brief name. Coarse but representative; the embedder doesn't need precision.
- Failure mode: any exception inside the RAG layer is swallowed so the prompt builder always produces a prompt.

### APN-11 — Supervisor hooks

- [backend/agents/supervisor.py](backend/agents/supervisor.py): right after `resolve_playbook`, if the playbook has `rag.enabled=True` and chromadb is available, call `ingest_source(project_dir, playbook_id, session_id=run_id or session_id)`. Emits an `rag_source_ingested` SSE event so the UI can show ingest progress.
- The same `session_id` is threaded into every `build_agent_prompt` call so per-session `__source` retrieval lines up with the data ingested at the top of the run.
- [backend/agents/migration_pipeline.py](backend/agents/migration_pipeline.py): new `_step_rag_learn` step appended after the round-trip step. Loads `field_parity.json` + `round_trip.json`, walks `file_map.json` `mappings[]`, reads each (source_root/src_path) and (converted_root/tgt_path), and calls `ingest_result` with `parity_pct` from the report. Caps at 200 rows to keep ingestion bounded. Skips `_disk_inferred` rows (no point storing rows the planner didn't author) and rows already marked `status=skipped`.

### APN-12 — RAG status / control endpoints

- [backend/main.py](backend/main.py): four new endpoints.
- `GET  /api/rag/status` — capability snapshot (chromadb installed?, env disabled?, available?, root path, collection inventory, embeddings provider, per-playbook enabled+agents+top_k).
- `POST /api/rag/seed` — rebuild `__curated` for every playbook from `backend/playbooks/examples/*.jsonl`. Idempotent.
- `POST /api/rag/seed/{playbook_id}` — rebuild one playbook.
- `POST /api/rag/clear-session/{session_id}` — drop the per-session `__source` collection for every playbook (cleanup hook).
- All four pass through the existing HTTP capture middleware so the Logs drawer shows their status + payload.

### APN-13 — Eval harness skeleton

- [backend/playbooks/rag/eval.py](backend/playbooks/rag/eval.py) — `python -m backend.playbooks.rag.eval --playbook <id> --queries <file.jsonl> --top-k 3`. Each line in the queries JSONL is shaped `{"query": "...", "must_match_path": "..."}`. Reports recall_pct = fraction of queries whose `must_match_path` appears in top-K hits. Not a benchmark — it's a smoke gate to run before flipping `rag.enabled` for a new playbook.
