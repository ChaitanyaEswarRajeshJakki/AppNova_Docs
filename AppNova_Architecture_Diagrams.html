LLM batch → master context doc → disk cache (Phase 3)

3B · Streaming Analysis Pipeline (POST /api/analyze/stream)

🖥 Frontend
POST with session_id, agent_ids, stack config
⏳ Wait Index
Poll until _indexing_status = "ready" (max 120 s)
🔎 RAG Search
ChromaDB search_code → shared code chunks + file summaries
🤖 Agent Loop
For each agent: rate-limit pacing → build prompt → LLM call
📡 SSE Events
start / thinking / complete / report / error / done
📊 AppContext
mergeAnalysisResult · updateAnalysisProgress · appendAgentThought
📑 ReportView
Render Mermaid + Markdown per agent tab

3C · Chat + CRAG Retrieval Flow (GET /api/chat/stream)

💬 User Message
ChatView sends message history + stack
🎯 Intent Detect
detect_chat_intent() → agent_id or "chat"
📥 Retrieve
search_code + search_analysis_results from ChromaDB
📐 Grade Context
LLM grades: relevant / partial / irrelevant
🔄 Rewrite Query
If partial/irrelevant: rewrite → re-retrieve (max 2 loops)
✍ Generate
ChatGroq / Gemini / Ollama streams response tokens
🖥 Stream → UI
SSE tokens → ThinkingBlock + ReactMarkdown rendering

ChromaDB Embed
Chunk 3000 chars / 300 overlap → jina-v2 CUDA encode → upsert
_indexing_status = "indexing" → "ready"
Phase 2 — BG
Repo Parse
Static AST: imports, symbols, component trees, entry points, circular deps → _repo_knowledge[id]
Phase 3 — BG
Context Ingest
LLM batch notes (60k char batches) → master context doc → uploads/{id}/codebase_context.json
Ready
Analysis & Chat
Full ChromaDB + repo knowledge + context cache available. User can analyze / chat.
1
Request arrives
POST /api/upload with files or GitHub URL. Thread-safe uuid4 session_id generated.
2
Files processed
ZIP extracted, encoding detected (chardet), language mapped, paths normalized. Binary/lock/map files skipped.
3
Immediate response
UploadResponse with session_id, file count, breakdown by language returned to frontend instantly.
4
BG Indexing starts
BackgroundTasks.add_task(_index_session_bg). Frontend polls /api/index-status/{id} every 2 s.
5
Index "ready"
_indexing_status updated to ready. Stream analysis waits for this before starting agents.
6
Agent loop
Sequential agents with rate-limit pacing. Gemini: full-codebase injection. Groq/Ollama: RAG chunks.
7
Reports ready
Results merged into AppContext, per-agent tabs populate with Markdown + Mermaid, DOCX generated.

status=indexing / phase=chunking → phase=encoding → phase=parsing → phase=building_context → status=ready / phase=done or status=error

8 · v2 Claude Code CLI Runner NEW · 2026-04-21

Subprocess-based DAG runner that replaces Sections 2–3 when the user hits /api/analyze-v2 or /api/chat

Each of the 12 agents runs as its own claude -p subprocess against the logged-in Claude Code Max subscription — no API keys consumed on the happy path. The supervisor layers agents into execution waves by their upstream dependencies; Wave 0 fans out up to 9 concurrent subprocesses.

🚀 agents/runner.py
Threaded subprocess.Popen. Works on any event loop. Windows .cmd shim bypassed via node.exe cli.js. 3-attempt transient retry. CREATE_NO_WINDOW.
🧭 agents/supervisor.py
compute_waves() topological sort. Wave 0 = 9 parallel · Wave 1 = code-gen · Wave 2 = testing + ui-ux. Per-cwd writer lock.
🔍 Discovery + briefs
One Haiku pass writes context/digest.md + brief_<agent>.md so each specialist starts with pre-distilled context.
📁 session_adapter.py
Mirrors uploads/{sid}/files/ → source/. Detects project root via manifest markers (package.json, pom.xml, .csproj, go.mod, ...).
💬 chat_bridge.py
Single-turn claude -p --output-format json. Read/Glob/Grep allowed, Write/Edit/Bash denied. Never raises → /api/chat falls back silently to the v1 chat_graph.
⚙️ config + prompts
AGENT_REGISTRY (12 specs) · HEAVY_MODEL (sonnet-4-6) + LIGHT_MODEL (haiku-4-5) · 24 h default timeouts matching Claude Code session ceiling.
💰 core/cost_tracker.py
Per-call SQLite ledger (data/cost_tracking.db) + openpyxl 5-sheet workbook. Subscription absorbs the actual spend; ledger reports API-equivalent virtual cost.
📋 v2_to_report_data.py
build_combined_markdown() flattens per-agent outputs for the existing tools/document_generator.generate_docx. No export-pipeline changes.
🔌 mcp_server/trace_server.py
Local stdio MCP server exposing trace SQLite. Register with claude mcp add appnova-traces so Claude Code sessions query backend telemetry directly.

Execution model — DAG waves

POST /api/analyze-v2
  │
  ├─▶ session_adapter.ensure_agent_layout()         ── uploads/{sid}/{source,converted,context}/
  │
  ├─▶ runner.run_discovery()                        ── 1× claude -p Haiku · ~2 min · writes digest.md + briefs
  │
  └─▶ supervisor.run_supervised()                   ── 3-wave DAG scheduler
        │
        ┌─ Wave 0 (parallel: 9 claude -p Sonnet subprocesses) ─────────────────────┐
        │  code-analysis · architecture · business-rules · security               │
        │  migration-planner · documentation · devops · data-migration            │
        │  integration                                                            │
        └─────────────────────────────────────────────────────────────────────────┘
        ▼
        ┌─ Wave 1 (1 claude -p Sonnet · writer) ────────────────────────────────────┐
        │  code-generation                              cwd=converted/ + --add-dir source/
        └───────────────────────────────────────────────────────────────────────────┘
        ▼
        ┌─ Wave 2 (parallel: 2 claude -p subprocesses) ────────────────────────────┐
        │  testing (haiku)               ui-ux (sonnet)                           │
        └─────────────────────────────────────────────────────────────────────────┘
        ▼
        v2_to_report_data.build_combined_markdown → generate_docx → exports/
SSE envelope · backward-compatible
The runner streams rich events (agent_start, assistant with tool_calls, tool_result, agent_complete) alongside the legacy {type: start|complete|error, agent_id, ...} shape. The existing React frontend (api.ts, AppContext.tsx) works unchanged — only one URL was flipped from /api/analyze/stream → /api/analyze-v2. · Full port log + 9 hotfixes: changes.md (2026-04-21 section).
