# 2026-06-05 17:15 — Two fixes: AuditLog 500 crash + nav click expands collapsed cards

## Problem

After a full run completes, `markStart` auto-collapses every card that is no
longer the active one — so at the end only the last-running agent's card is
expanded. Clicking a nav item in the sidebar only *scrolled* to the card
(via `scrollIntoView`) but never removed the `collapsed` class. The user saw a
thin header bar with a Done pill and no report body — "nothing shown."

## Fix — `frontend/app.js` (nav click handler, ~line 1246)

Before:

```js
const card = $(`card-${li.dataset.id}`);
if (card) card.scrollIntoView({ behavior: 'smooth', block: 'start' });
```

After:

```js
const card = $(`card-${li.dataset.id}`);
if (!card) return;
if (card.classList.contains('collapsed')) {
  card.classList.remove('collapsed');
  const tog = card.querySelector('.toggle-btn');
  if (tog) tog.textContent = '▾';
}
card.scrollIntoView({ behavior: 'smooth', block: 'start' });
```

## Fix 2 — `backend/main.py:3146` (`_run_analysis_stream`)

`POST /api/run-selected/{sid}` (and `/api/analyze`, `/api/resume`) was returning
HTTP 500 before the SSE stream even opened:

```text
TypeError: AuditLog.session_start() missing 1 required positional argument: 'user'
```

The `user` argument (`_claims_owner(authorization)`) is not in scope inside
`_run_analysis_stream` — the function receives only `session_id`. Auth is already
enforced by middleware before reaching this code, so passing `""` is safe and
consistent with `AuditLog.session_end`'s existing `user=""` default.

**Fix:** `AuditLog.session_start(session_id, "", target_stack=..., agent_ids=...)`

Clicking any sidebar item now expands the target card (if collapsed) before
scrolling — the report body is immediately visible. Cards that are already
expanded pass through unchanged.
