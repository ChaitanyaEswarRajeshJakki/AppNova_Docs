# 2026-06-09 14:50 — Fix report table last-column clipping

## Problem
In rendered reports (e.g. the migration-planner "Current→Target Component Map"
table), the final column ("Target-stack rationale") was cut off at the right
edge and showed no data. Cells contain `<code>` file paths like
`src/app/services/total_booking.service.js` — long, unbreakable tokens. With
`width: 100%` auto-layout tables and no word-wrapping, the intrinsic table width
exceeded the page/container width, so the last column was pushed off-screen and
clipped (no horizontal scroll on the report container).

## Fix
Made table cells wrap so the table always stays within its container width.

- `backend/agents/tools/export.py` (`_STANDALONE_CSS`, used by HTML + PDF export):
  - `table`: added `max-width: 100%` and `table-layout: fixed`.
  - `th, td`: added `overflow-wrap: anywhere; word-break: break-word;`.
- `frontend/style.css` (`.final-report` in-app report view):
  - `table`: added `max-width: 100%` and `table-layout: fixed`.
  - `th, td`: added `vertical-align: top; overflow-wrap: anywhere; word-break: break-word;`.

`table-layout: fixed` constrains columns to the available width and distributes
them; `overflow-wrap/word-break` lets long code paths break instead of forcing
horizontal overflow. The DOCX export path is unaffected (Word tables auto-wrap).
