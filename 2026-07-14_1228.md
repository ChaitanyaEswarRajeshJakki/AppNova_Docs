# 2026-07-14 12:28 — Clean up DOCX/PDF layout (page splits, spacing, table alignment)

## Why
User reported the exported DOCX/PDF had messy line/page breaking and wanted each
topic split and aligned cleanly.

## What was wrong
1. **Duplicate title** — the build passed `--metadata title` *and* the Markdown starts
   with an `# H1`, so pandoc rendered the title twice (visible title block + body H1).
2. **Duplicate table of contents** — `--toc` added an auto-TOC on top of the manual
   "Contents" section already in the document.
3. **Mid-word line breaks in wide tables** — the PDF CSS used `overflow-wrap: anywhere`,
   which let the squeezed 5-column guardrail table break words mid-character
   ("Governan/ce", "Develop/ment").
4. No enforced page breaks between Parts/Topics/Chapters; default Word spacing.

## Fixes (regenerated the two deliverables at the external location)
- **PDF (CSS `handbook_style.css` → Chromium print):**
  - `overflow-wrap: break-word; word-break: normal; hyphens: none` → words wrap only at
    boundaries; columns size to whole words.
  - `break-before: page` on every Part (`h1`) and every Topic/Chapter (`h2[id^=topic-/chapter-]`);
    title exempted so page 1 isn't blank.
  - `break-after: avoid` to keep headings with following text; `tr { break-inside: avoid }`
    and `thead { display: table-header-group }` so rows don't split and headers repeat.
  - Roomier spacing (line-height 1.55, `@page` margins 22/18/20/18 mm), zebra-striped rows,
    narrow no-wrap checkbox column, page-number footer.
  - Dropped the duplicate title/TOC (`pagetitle` only; no `--toc`).
- **DOCX (pandoc):**
  - New Lua filter `pagebreaks.lua` — inserts a real Word page break before each
    Part/Topic/Chapter (16 total) and zeroes table colspec widths so Word auto-fits columns.
  - Styled `reference.docx` (via python-docx) — orange headings to match the PDF,
    1.15 line spacing, 6 pt paragraph spacing.
  - Dropped `--metadata title` (no duplicate) and `--toc` (kept the manual Contents).

## Verification
- Rasterized PDF pages with PyMuPDF and inspected: single title on p1; guardrail wide table
  wraps cleanly (no mid-word breaks); 2-col topic tables and role/checklist tables aligned;
  44 pages, each Part/Topic/Chapter starts on its own page.
- DOCX XML checked: title string appears once; 16 page breaks; all 24 tables
  `w:tblW type="auto"` (autofit to contents).
