AppNova AI

Complete System Architecture - 7 Layers + 30 Agents + 6 Phases · + v2 CLI runner (2026-04-21)

NEW · v2 Claude Code CLI runner · subscription primary, LLM cascade fallback

/api/analyze-v2 and /api/chat run each agent as a claude -p subprocess under your Claude Code Max subscription. The 7-layer architecture below still describes the v1 (LangGraph) path — it's the automatic fallback when the v2 subprocess path fails. See Layer 8 at the bottom for the v2 runner details, or changes.md (2026-04-21) for the full port log + 9 hotfixes.

8
v2 Claude Code CLI Runner (NEW)
subscription · subprocess · DAG waves
Replaces the LangGraph orchestration in Layer 4 when the user hits /api/analyze-v2 or /api/chat. Each agent is an independent claude -p subprocess run under the logged-in Claude Code Max subscription. The 4-provider LLM cascade in Layer 5 becomes the fallback path only — triggered when the CLI is unreachable, rate-limited, or returns an error. See changes.md (2026-04-21) for the port log + 9 hotfixes.
Runner (subprocess)
agents/runner.py
subprocess.Popen + 3 daemon threads (stdin / stdout / stderr). Stream-json lines fed back to the asyncio event loop via loop.call_soon_threadsafe. Works on both WindowsProactorEventLoop and WindowsSelectorEventLoop (event-loop agnostic). Windows .cmd shim bypassed via node.exe cli.js.
Supervisor (DAG waves)
agents/supervisor.py
compute_waves() topologically layers the 12 agents by their AgentSpec.upstream. Wave 0 = 9 parallel; Wave 1 = code-generation; Wave 2 = testing + ui-ux. Per-cwd lock keeps writers serialised on converted/.
Discovery pass
agents/runner.py::run_discovery
One cheap Haiku claude -p call before the waves — writes context/digest.md + context/brief_<agent>.md per specialist so downstream agents skip re-exploration.
Session adapter
agents/session_adapter.py
Mirrors uploads/{sid}/files/ → uploads/{sid}/source/ and detects the project root (package.json, pom.xml, composer.json, .csproj, go.mod, ...). Creates converted/ + context/.
Chat bridge
agents/chat_bridge.py
Single-turn claude -p --output-format json wrapper. Read/Glob/Grep allowed; Write/Edit/Bash/WebFetch/Task denied. Never raises — returns (text, error_or_None) so /api/chat can fall back to Layer 4+5 transparently.
Cost ledger
core/cost_tracker.py + model_pricing.yaml
Per-agent SQLite ledger (data/cost_tracking.db) capturing input/output/cache tokens + CLI-reported total_cost_usd. Subscription absorbs the real spend; ledger shows API-equivalent virtual cost for reporting via openpyxl 5-sheet workbook.
Report combiner
agents/v2_to_report_data.py
build_combined_markdown() concatenates per-agent markdown results into one string for document_generator.generate_docx / generate_pdf. No changes to the export pipeline (Layer 7) required.
MCP trace server
mcp_server/trace_server.py
Local stdio MCP server exposing the trace SQLite read-only. Register via claude mcp add appnova-traces -- python -m mcp_server.trace_server so Claude Code CLI sessions can query backend telemetry directly.
