# 2026-06-09 13:35 — Fix scaffold crash on nested-.csproj layouts (relative vs absolute path)

## Problem
On the .NET conversion (`converted/backend/TotalBookingAI.API/...`), the scaffold step threw:

```
ValueError: '...\uploads\3dfca6fbea95\converted\backend\TotalBookingAI.API' is not in the
subpath of 'uploads\3dfca6fbea95\converted' OR one path is relative and the other is absolute.
```

at `scaffold.py:937` (`_write_split_run_scripts`) and `scaffold.py:141` (`ensure_mandatory_structure` nested-csproj backstop).

## Root cause
`converted_dir` is passed in **relative** (`uploads/<sid>/converted`), but the API project dir
(`nested_api_dir` / `api_project_dir` / `frontend_dir`) comes from a glob that yields **absolute**
paths. `Path.relative_to()` raises on a relative-vs-absolute mix. Effect: the split run-script
generator and the nested-csproj backstop never completed for nested .NET layouts, so `run.sh` /
`run.bat` / `run_backend.cmd` weren't written correctly. Non-fatal (caught + logged) but the run
scripts for the converted app were missing/wrong.

## Changes — `backend/agents/tools/scaffold.py`
Resolve **both** operands before `relative_to` (mirrors `file_coverage.py::_rel_posix`):
- L141 `ensure_mandatory_structure` nested-csproj backstop — `nested_api_dir.resolve().relative_to(converted_dir.resolve())`.
- L937/940 `_write_split_run_scripts` — `api_project_dir` + `frontend_dir` resolved against `converted_dir.resolve()`.
- L1210 `_ensure_dotnet_appsettings` prefix — was guarded by `except ValueError → ""` (silently produced a wrong empty prefix); now resolves both sides so the created-file prefix is correct.

## Verification
- `scaffold.py` parses clean.
- Reproduced the exact failing paths: OLD `relative_to` → `ValueError`; NEW `resolve().relative_to(resolve())` → `backend/TotalBookingAI.API`. ✅

## Note
This is independent of the file-map recovery work (changes/2026-06-09_1320.md) but matters for the
same session: when code-generation is re-run, the scaffold step runs again — this fix lets the
nested-.NET run scripts be written instead of crashing the backstop.
