## Repository fixes required for the lists to return data on SQLite (pre-existing bugs,
## only surfaced once rows existed)
- `Repositories/TotalBooking/TotalBookingRepository.cs`
  - AllAsync raw SQL was SQL-Server-only (`GETDATE()`, `OFFSET..FETCH`). Now picks the
    dialect via `_db.Database.IsSqlite()` → `datetime('now')` + `LIMIT 200` on SQLite,
    original syntax on SQL Server.
  - ApplyJudgeCanEditFlags used `dynamic` member access with snake_case names against a
    typed TotalBooking entity (threw RuntimeBinderException: `judge_review_end_time`).
    Rewrote to typed property access and write the computed judge flags into the output
    dictionary instead of onto the entity.
- `Repositories/PCDEC/PcDeclarationRepository.cs`
  - Role-count raw SQL referenced legacy `role_id`/`user_id`; the EF-generated role_user
    pivot uses `RolesId`/`UsersId`. Fixed both queries and aliased the scalar as `"Value"`
    so `SqlQueryRaw<int>` can read it.
  - All three list branches applied `.Include(...)` after `.Select(new PcDeclaration{...})`
    (EF: "Include after Select" InvalidOperationException). Moved Where/OrderBy/Take before
    the projection and populate WorkflowStep/ArrestAgency/ArrestOfficer inside the Select.
