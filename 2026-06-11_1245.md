## The tee primitive (and why it's exactly this)
`powershell -NoProfile -Command "$fs=[IO.FileStream]::new('<log>',Create,Write,
ReadWrite); $sw=[IO.StreamWriter]::new($fs,UTF8Encoding($false)); $sw.AutoFlush=$true;
$input | %{ [Console]::WriteLine($_); $sw.WriteLine($_) }; …"`
- **FileShare.ReadWrite** — browser_test / the run-manager tailer can read the log
  WHILE the server writes it. A plain `Out-File` / `Tee-Object` holds an EXCLUSIVE
  lock → concurrent reads fail (verified: the read threw a sharing violation).
- **UTF8Encoding($false)** — clean UTF-8, no BOM; browser_test reads as UTF-8.
  (`Tee-Object -Encoding` is PS 6+ only — it ERRORS on Windows PowerShell 5.1; a
  bare Tee-Object writes UTF-16 garbage.)
- **AutoFlush** — readers see the latest lines immediately.
- `2>&1` before the pipe merges stderr at the cmd level so PowerShell sees plain
  text (no NativeCommandError wrapping).
