✨ Flow 4 — Claude Code CLI Runner (v2, 2026-04-21) · /api/analyze-v2

Replaces the LangGraph orchestration in Flow 1 with a subprocess-based DAG wave scheduler that calls your Claude Code Max subscription. Each of the 12 agents runs as its own claude -p process with cwd=uploads/{sid}/source/, streaming stream-json tool-call events back to the frontend via the same SSE envelope as Flow 1.

What changes vs Flow 1:
• Auth: Max subscription (no API keys).
• Context: each agent gets the whole source/ as cwd with Read/Glob/Grep — no RAG chunking.
• Parallelism: 9 agents in wave 0, then 1, then 2 — ~3× faster than sequential LangGraph nodes.
• Fallback: none at the runner level — if Claude Code fails, the error surfaces. /api/chat bridge DOES fall back to Flow 2's chat_graph on error.
• Modules: backend/agents/runner.py · supervisor.py · state.py · prompts.py · session_adapter.py · v2_to_report_data.py.
• Full port log + 9 hotfixes: changes.md (2026-04-21 section).
