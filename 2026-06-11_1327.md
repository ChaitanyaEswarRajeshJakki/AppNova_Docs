## Root cause

1. `AppRouter.tsx` `AppLayout` (the authenticated shell) was a bare
   `<Suspense><Outlet/></Suspense>` — no navbar/header was ever built, so every
   authenticated page lost its navigation.
2. `DashboardPage.tsx` was a placeholder stub (heading + welcome line + a
   misplaced day/night toggle), not the legacy launchpad.
3. Static image assets were un-ported: the uploaded source omitted `src/images/`,
   so `public/assets/` shipped empty and every `/assets/images/...` reference 404s.
4. `BookingListPage.tsx` imported `./components/TotalBookingListJudge` /
   `TotalBookingListOfficer`, but the files on disk are `BookingListJudge.tsx` /
   `BookingListOfficer.tsx` — a hard `vite build` failure masked by dev-server
   lazy-loading.
