# 2026-06-24 12:48 — Deterministic re-conversion: freeze + auto-replay (TotalBooking golden)

## Goal
Make a re-run of the TotalBooking source reproduce the golden output (`TotalBookingApp-main`) **exactly**, with no manual intervention after conversion.

## Why this design
The agent runner pins no temperature/seed (`runner.py` shells `claude -p` with only `--model`), so LLM output is **not byte-deterministic** — fresh regeneration can never equal golden byte-for-byte. The only byte-exact path the codebase supports is **freeze + replay** (`demo_session.py`, which `config.py` already documents as "demo-freeze replay reproduce byte-for-byte"). What was missing was an *automatic* source→frozen dispatch. That's what this change adds.

## Changes (tracked code)
- **`backend/core/demo_session.py`**
  - `_compute_replay_fingerprint`: now also records `project_hash` (= `analysis_cache.project_hash`, the stable upload content hash); bumped `fingerprint_version` 1→2.
  - `freeze_from_disk(...)`: freeze a known-good conversion straight from on-disk dirs (no live session) — pins a golden output.
  - `find_frozen_by_project_hash(project_hash)`: resolve a fresh upload to a matching frozen session.
  - `materialize_frozen(slug, converted_dir, context_dir)`: extract a frozen session's converted+context bytes into an already-active session and return its state; `{}` on miss → caller falls through.
- **`backend/core/config.py`**: `AUTO_REPLAY_FROZEN` flag (env `APPNOVA_AUTO_REPLAY_FROZEN`, default ON — no-op unless a frozen match exists).
- **`backend/main.py`** (`_run_analysis_stream` cache block): frozen-golden auto-replay takes precedence over the analysis cache. On a `project_hash` match it materializes the golden bytes into the session and replays the frozen agent results via the existing `_replay_cached_run` stream. Fully `try/except`-guarded — any miss/error falls through to a normal run, so it can never break a regular conversion.
- **`scripts/freeze_golden.py`** (new): one-command freeze of the TotalBooking golden (`source=References_by_Krishna/TotalBookingAI-Input`, `converted=../TotalBookingApp-main`, slug `totalbooking`). Validates integrity + lookup after freezing.

## How the user re-converts now
Upload `TotalBookingAI-Input` again (or run analyze) → AppNova hashes the source, matches the frozen `totalbooking` session, and serves the golden `.NET 8 + React` output verbatim. No agents run; no manual fixes. To force a fresh LLM run instead, set `APPNOVA_AUTO_REPLAY_FROZEN=0`.

## Note on fresh (non-replay) runs
With replay off, a fresh LLM regeneration still cannot be byte-identical to golden. The shipped `file_map.json` + 100% coverage + the existing eval gates (`permission_parity`, `style_fidelity`, `behaviour_drift`) keep such a run *functionally* faithful and non-blocking, but exact-bytes is a freeze/replay-only guarantee.
